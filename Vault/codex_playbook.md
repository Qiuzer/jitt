# Codex 工程级操作手册

## 核心原则

1.  steer 只改局部，不改方向\
2.  大改动一律重开，不修补\
3.  queue 用来串任务，不用来修 bug\
4.  AI 默认是"追加"，不是"重构"

------------------------------------------------------------------------

## 标准工作流

### 阶段 1：定义需求

-   明确技术栈、结构、风格
-   一次生成接近最终形态

### 阶段 2：生成中（steer）

✅ 小修改（变量、日志、错误处理）\
❌ 架构/范式修改

### 阶段 3：生成后检查

-   冗余代码
-   未使用函数
-   风格一致性

### 阶段 4：重构

-   删除冗余
-   统一风格
-   合并逻辑

------------------------------------------------------------------------

## steer / queue 策略

### steer 模板

修改以下内容： - 仅修改 XXX - 不要改变整体结构 - 删除旧实现（如果有）

### queue 使用

用于串联任务： - 写接口 → 写测试 → 写文档

❌ 不用于修 bug

------------------------------------------------------------------------

## 防止屎山代码技巧

1.  强制删除旧代码\
2.  强制统一风格\
3.  禁止多方案并存\
4.  控制文件粒度\
5.  明确生产级目标

------------------------------------------------------------------------

## 实战流程

1.  生成代码\
2.  steer 小修\
3.  检查\
4.  重构\
5.  queue 扩展

------------------------------------------------------------------------

## 进阶认知

-   AI 不会自动重构
-   中途 steer 会增加技术债
-   prompt = 架构设计

------------------------------------------------------------------------

## 总结

前期定义清楚 → 中期小修 → 后期重构 → queue 串流程
